使用Hugo改动原有博文内容 @ UsAndThem26 | Sunday, Jan 16, 2022 | 1 | Sunday, Jan 16, 2022

「单纯想要修改原先的的文章，只要目的达到了就可以了。于是就挨个试试看，有时间再去了解原理吧。」

一、在vscode打开需要修改的文章.md文件

     * 修改内容并点击保存。

二、打开终端

 根目录myblog下

 hugo server -t dream --buildDrafts

     "Start building sites … 
     hugo v0.91.2+extended darwin/arm64 BuildDate=unknown
     ...
     Running in Fast Render Mode. For full rebuilds on change: hugo server --disableFastRender
     Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)
      Press Ctrl+C to stop"
 ^C%                             
 
 cd public

 git push -u origin master

     "分支 'master' 设置为跟踪来自 'origin' 的远程分支 'master'。
     Everything up-to-date"

 cd ..

     * 回到根目录

 hugo --theme=dream --baseUrl="https://usandthem26.github.io/" --buildDrafts

     "Start building sites …
     ...
     Total in 29 ms"

 cd public

 git init

     "已重新初始化已存在的 Git 仓库于 /Users/juanjuanli/myblog/public/.git/"

 git add .

 git commit -m "第一次提交"

     "...

     1 file changed, 1 insertion(+), 1 deletion(-)"

     * 到这里直接

 git push -u origin master
 
     * 显示

     【【   枚举对象中: 9, 完成.
     对象计数中: 100% (9/9), 完成.
     ...
     分支 'master' 设置为跟踪来自 'origin' 的远程分支 'master’。】】
 
     * 但更新不成功。

     * 继续
 
 git remote add origin https://github.com/usandthem26/usandthem26.github.io.git

     * 提示"error: 远程 origin 已经存在。"

     * 再

 git push -u origin master

     "分支 'master' 设置为跟踪来自 'origin' 的远程分支 'master'。
     Everything up-to-date"

     * 就成功了

三、刷新网页，原博文更改完成
